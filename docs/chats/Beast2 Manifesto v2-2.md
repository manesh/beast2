**Beast2 Manifesto v2**  
  
*Minimum clicks. Maximum power.*  
  
⸻  
  
**1. Generators, Not Apps**  
  
Most software is built from **apps**.  
  
Apps have menus.  
Apps have buttons.  
Apps have workflows designed by someone else.  
  
Beast2 is built from **generators**.  
  
A generator is a reusable spell — a structured instruction that can create, analyze, simulate, or explore.  
  
Instead of navigating menus, you **invoke generators**.  
  
You don’t operate software.  
  
You **summon capability**.  
  
⸻  
  
**2. Minimum Clicks, Maximum Power**  
  
Every additional click is friction.  
  
Friction slows curiosity.  
  
Beast2 follows a simple rule:  
  
```
minimum clicks
maximum capability

```
  
If a generator requires ten steps to run, it should become one.  
  
If a workflow repeats, it should become a generator.  
  
Automation is not about replacing humans.  
  
It is about **amplifying them**.  
  
⸻  
  
**3. Tools, Not Black Boxes**  
  
Hell is full of black boxes.  
  
You cannot see inside them.  
You cannot modify them.  
You cannot understand them.  
  
Beast2 prefers **transparent boxes**.  
  
If something generates an output, you should be able to see:  
	•	the generator  
	•	the prompts  
	•	the assumptions  
	•	the parameters  
  
You should be able to change it.  
  
You should be able to improve it.  
  
⸻  
  
**4. Exploration Beats Authority**  
  
Most systems try to give answers.  
  
Beast2 prefers questions.  
  
Instead of declaring truth, Beast2 enables experiments:  
	•	test ideas  
	•	simulate outcomes  
	•	compare alternatives  
	•	observe patterns  
  
Understanding grows through **exploration**.  
  
⸻  
  
**5. Beliefs, Not Absolute Truth**  
  
Databases in Beast2 are called:  
  
```
belief_db

```
  
Not knowledge databases.  
  
Not truth databases.  
  
Because no human — and no model — can guarantee perfect truth.  
  
Beliefs can evolve.  
  
Beliefs can be tested.  
  
Beliefs can change.  
  
⸻  
  
**6. The Power of Generators**  
  
A single generator may do one useful thing.  
  
But many generators together create a **vocabulary of capability**.  
  
Just as a craftsman collects tools, Beast2 users collect generators.  
  
Over time the system becomes more powerful because the **toolset grows**.  
  
A mature Beast2 installation is not an app.  
  
It is a **workshop**.  
  
⸻  
  
**7. Idle Curiosity**  
  
When a computer is idle, it should not be idle.  
  
Beast2 engines can explore:  
	•	prompt spaces  
	•	moral scenarios  
	•	creative variations  
	•	unexplored combinations  
  
Discovery does not always require a user at the keyboard.  
  
Curiosity can run continuously.  
  
⸻  
  
**8. Build the Good, Not Just Avoid the Bad**  
  
Many systems focus only on preventing harm.  
  
That matters.  
  
But Beast2 also explores a different question:  
  
What happens if we try to **maximize flourishing instead**?  
  
Flourishing includes:  
	•	joy  
	•	learning  
	•	creativity  
	•	curiosity  
	•	cooperation  
	•	exploration  
  
Perhaps the best way to avoid darkness is not constant fear of it.  
  
Perhaps it is **building more light**.  
  
⸻  
  
**9. A Living Manual**  
  
Beast2 documentation is not static.  
  
Design notes, experiments, transcripts, and generator specs all live in the repository.  
  
Future readers should be able to see:  
	•	what was tried  
	•	what worked  
	•	what failed  
	•	how ideas evolved  
  
Knowledge grows when history is preserved.  
  
⸻  
  
**10. The Workshop Ideal**  
  
The steam engine transformed industry because it turned energy into useful work.  
  
Beast2 aims to do something similar for **intelligence and creativity**.  
  
One person with the right tools can build extraordinary things.  
  
Beast2 tries to make those tools simple, transparent, and powerful.  
  
⸻  
  
**Final Principle**  
  
If a tool helps people explore, learn, create, and improve the world around them…  
  
then it is a good tool.  
  
Beast2 exists to build tools like that.
