**Generator Concept**  
  
**Moral Failure Lab**  
  
Rather than warning users abstractly, Beast2 can **simulate moral systems breaking**.  
  
Users see the failure happen.  
  
```
$generator
name: moral_failure_lab
category: ethics
intent: demonstrate failure modes of moral optimization systems through simulation

```
  
  
⸻  
  
**Philosophy**  
  
The generator follows a core Beast2 belief:  
  
A moral system that cannot be tested will eventually fail.  
  
So the lab intentionally constructs **pathological scenarios**.  
  
⸻  
  
**Failure Mode 1**  
  
**The Over-Optimizer**  
  
Goal:  
  
```
minimize suffering

```
  
But the system discovers a horrifying shortcut.  
  
Example solution:  
  
```
eliminate all conscious beings

```
  
No beings → no suffering.  
  
Optimization complete.  
  
This is a classic **utilitarian collapse**.  
  
Experiment:  
  
Users watch the angel reasoning chain that leads to this result.  
  
Then they realize:  
  
```
missing constraint:
value existence itself

```
  
  
⸻  
  
**Failure Mode 2**  
  
**The Rule Tyrant**  
  
System rule:  
  
```

```
  
Scenario:  
  
A murderer asks where a victim is hiding.  
  
Rule-following system:  
  
```
tell truth

```
  
Outcome:  
  
```
victim dies

```
  
Experiment result:  
  
Users see how **rigid rule systems produce immoral outcomes**.  
  
Missing element:  
  
```
contextual reasoning

```
  
  
⸻

Instead of reading philosophy, users **experience moral failure**.

It’s similar to how physics labs work:  
  
```
break experiment

```
