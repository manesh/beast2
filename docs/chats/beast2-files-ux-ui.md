beast2-files-ux-ui.md

laws
- every writeable file can be revealed or changed through the ui
- every folder where beast2 writes files or a user is expected to write files can be revealed in file browser by clicking

first run ui
- user is prompted where to save beast2 files ("default ok?", including startup config files, master key-value store, database of databases)

known beast2 files:
- startup config file (technical, max memory usage for warm)
- user config file (font/ui scale)
- master database file, beast2.db (saved location of all databases and related directories, key-value store of anything app might need)

gallery ui
- user can open a folder to view files just like system file browser
- videos autoplay and loop, up to 10s, videos of longer duration display a play button

gallery ui: beast2 generated
- beast2 generated files have both the image/video/text as well as an associated db with additional data which shows in overlays and on the sidebar
- alternatively viewing a beast2 generated file with embedded data will decode and display that data


gallery ux: view modes
- folder
- db
- combined folders/dbs (check/uncheck from master list)
