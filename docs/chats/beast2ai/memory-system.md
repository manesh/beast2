# MEMORY_SYSTEM.md
> Memory is testimony, not truth.
>
> Every memory should have provenance.
>
> Every memory should be reversible.
---
# Purpose
The Beast2 Memory System exists to provide durable, inspectable, and versioned memory for humans and AI agents.
The objective is not perfect recall.
The objective is:
- useful recall
- explainable recall
- reviewable recall
- portable recall
---
# Core Principles
## Memory Is Testimony
Memories are not facts.
Memories are claims about prior events.
A memory may be:
- correct
- incomplete
- outdated
- mistaken
The system should preserve uncertainty when appropriate.
---
## Memory Requires Provenance
Every memory should answer:
```text
Why do I believe this?
```
Examples:
```text
User statement
Agent observation
Project document
System event
```
Unknown provenance should be explicitly marked.
---
## Memory Must Be Reversible
No memory should become permanent without history.
Users should be able to:
- inspect
- diff
- review
- rollback
memory changes.
---
# Memory Layers
## Raw Events
Lowest-level record.
Examples:
```text
chat messages
tool calls
agent actions
system events
```
Raw events are append-only.
---
## Session Summaries
Generated summaries of individual sessions.
Examples:
```text
work completed
decisions made
questions raised
```
Session summaries may be generated by agents.
---
## Topic Memories
Long-lived memories organized by subject.
Examples:
```text
Beast2
Heavenfleet
AI Research
Hosting
Philosophy
```
Topic memories are the primary retrieval target.
---
## Executive Summaries
Highly compressed summaries.
Examples:
```text
current goals
major decisions
important relationships
```
Used when context budgets are limited.
---
# Memory Structure
Suggested layout:
```text
memory/
    user.md
    projects/
        beast2.md
        heavenfleet.md
    topics/
        ai.md
        philosophy.md
        hosting.md
    summaries/
        executive.md
    sessions/
        2026-06-18.md
        2026-06-19.md
```
Implementation details may evolve.
---
# Versioning
Git is the preferred backend.
All memory changes should be versioned.
Every change should include:
```text
author
timestamp
reason
```
when available.
---
# Memory Proposals
Agents should not silently modify canonical memory.
Preferred workflow:
```text
Agent
    ↓
Memory Proposal
    ↓
Review
    ↓
Approval
    ↓
Commit
```
---
# Memory Metadata
Memory entries may include:
```yaml
statement: Canonical domain is heavenfleet.org
source: User statement
author: Michael
confidence: 1.0
created: 2026-06-18
```
Metadata should remain human-readable.
---
# Memory Compression
Memory grows continuously.
Compression is required.
Compression should:
- reduce redundancy
- preserve decisions
- preserve relationships
- preserve provenance
Compression should not destroy traceability.
---
# Memory Pyramid
```text
Raw Events
    ↓
Session Summaries
    ↓
Topic Memories
    ↓
Executive Summaries
```
Higher levels are increasingly compressed.
Lower levels remain available for verification.
---
# Memory Retrieval
Retrieval should answer:
```text
What should be loaded now?
```
Retrieval should prioritize:
- relevance
- recency
- importance
over:
- total volume
---
# Memory Quality
Good memories are:
- relevant
- concise
- attributable
- useful
Bad memories are:
- duplicated
- speculative
- contradictory
- unsupported
Memory quality matters more than memory quantity.
---
# Hallucination Resistance
Potentially incorrect memories should be detectable.
The system should support:
- version history
- diffs
- provenance inspection
- rollback
- review workflows
Agents may be wrong.
Memory should remain auditable.
---
# Memory Inspection
Users should be able to ask:
```text
Why do you remember this?
```
The system should provide:
- source
- author
- timestamp
- related memories
when available.
---
# Long-Term Goal
The memory system should survive:
- model replacement
- vendor replacement
- hardware replacement
- software replacement
The objective is durable knowledge.
Models are temporary.
Memory is infrastructure.
---
# Final Rule
Never silently rewrite history.
Create a new memory.
Version the change.
Preserve the old one.
