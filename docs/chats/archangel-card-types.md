Your current structure is strong because it separates:

* the core claim
* the reasoning
* the response to objections
* the evidence

That already feels closer to a courtroom / encyclopedia hybrid than a social feed.

I think the key is:

every card type should answer a specific question.

Then a claim page becomes composable and readable.

Here are the card types I think fit  archangel.my￼ best.

⸻

Core canonical cards

claim

The central statement being asserted.

claim
-archangel.my can help build a better civilization

⸻

summary

Ultra-short version.

Good for previews/search/mobile.

summary
-public memory systems reduce manipulation by preserving testimony and dissent.

⸻

argument

Positive reasoning supporting the claim.

argument
-public memory makes deception harder because contradictions remain visible over time.

Could allow multiple argument cards.

⸻

address-counter-claim

Directly responds to an objection.

This is excellent because it acknowledges opposition without requiring a separate page jump.

address-counter-claim
-this could become another biased platform.
-preserving dissent and edit history reduces that risk.

⸻

evidence

Raw supporting material.

evidence
-image
-caption: timeline showing public contradictions

or

evidence
-video
-caption: interview clip

* poll

* arbitrary formatting
* reputation systems initially

dense, structured, scrollable records.

Not infinite discourse trees.
