is-it-beastly-enough.md

purpose

this document defines a guideline for contributors to beast

not a checklist
not a gate
not a rule system

a question:

is it beastly enough

⸻

opening

legendary

in Dwarf Fortress:

a legendary artifact is:
	•	made by a master
	•	sometimes made by a novice
	•	given a name
	•	given a story
	•	engraved with meaning
	•	indestructible
	•	remembered

it lasts

⸻

beast standard

beast does not optimize for:

accepted

beast asks:

is it worthy

not just:
	•	does it work
	•	does it pass tests
	•	does it meet requirements

but:
	•	does it matter
	•	does it endure
	•	does it elevate the system

⸻

core questions

before contributing, ask:

1. do you love it

do you want this for yourself
would you use it daily
would you miss it if it did not exist


⸻

2. does it solve a real problem

#axiom

when you solve your own problem
you solve everyone's problem

not hypothetical
not imagined

real

⸻

3. is it clear

can someone understand what this does
can they inspect it
can they trust it

no hidden behavior
no magic

⸻

4. is it necessary

should this exist
or is it adding noise

does it:
	•	remove work
	•	reduce lag
	•	simplify systems

⸻

5. is it fast

does it block the user
does it introduce lag

beast does not tolerate unnecessary delay

⸻

6. is it correct

does it do exactly what it claims

not:
	•	"close enough"
	•	"probably fine"

but:
	•	precise
	•	deterministic
	•	reproducible

⸻

7. would you reward it

would you publicly award someone a trillion halos for this

not politely

not out of obligation

but because:

this is exceptional


⸻

8. is it legendary

would this stand the test of time
would this still make sense in 10 years
would someone study this and learn from it

does it feel:
	•	crafted
	•	intentional
	•	complete

⸻

what is not beastly enough

examples:
	•	unclear behavior
	•	hidden side effects
	•	unnecessary abstraction
	•	slow feedback loops
	•	features without purpose
	•	solutions you would not use yourself

⸻

what is beastly

examples:
	•	simple, obvious solutions
	•	visible systems
	•	fast feedback
	•	reproducible outputs
	•	tools that feel alive
	•	tools that remove friction
	•	tools that teach

⸻

contribution philosophy

beast is not a collection of features

beast is:

a system of artifacts that persist
a system that improves over time
a system that does not forget

every contribution becomes part of that system

⸻

approval

this document is not absolute

it is a guideline

if you believe your contribution is:
	•	useful
	•	clear
	•	necessary
	•	and possibly legendary

submit it

who can say what will meet these criteria

⸻

final principle

do not aim for acceptance
aim for something you are proud to stand behind


⸻

closing

build something worthy

or do not build it
